# Repository Map

This repository follows a harness-engineering style knowledge base.

Start here:

- `docs/index.md`: top-level map
- `docs/project-manager/`: project framing, responsibilities, milestones, risks
- `docs/hardware/`: hardware architecture, BOM, protocols, tradeoffs
- `docs/design-doc/`: design-document drafting notes and figure plan
- `design_document.tex`: main LaTeX draft

Working rules:

- Treat `docs/` as the system of record for durable project knowledge.
- Prefer small focused files over one long note.
- Use progressive disclosure: begin with an index, then follow links to details.
- Update the relevant note when decisions change, especially interfaces, the hardware model, budget, or verification criteria.
